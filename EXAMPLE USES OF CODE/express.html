Express is a minimal, unopinionated web framework for Node.js used to build servers and APIs.
It sits on top of Node's built-in HTTP module and adds routing, middleware support, and utilities that make building web applications much easier.

Setup:
To use Express, first initialise a Node project and install the package:

npm init -y
npm install express

Basic Server Example:
const express = require('express');
const app = express();

// Middleware to parse JSON request bodies
app.use(express.json());

// GET route
app.get('/', (req, res) => {
  res.send('Hello, World!');
});

// GET with URL parameter
app.get('/users/:id', (req, res) => {
  const { id } = req.params;
  res.json({ userId: id, name: 'Alice' });
});

// POST route
app.post('/users', (req, res) => {
  const { name, email } = req.body;
  res.status(201).json({ message: 'User created', name, email });
});

// Start the server
app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});

Key Concepts:
express() — Creates an app instance. This is the starting point of every Express application.
app.use() — Registers middleware that runs on every request before the route handler is reached.
app.get() / app.post() / app.put() / app.delete() — Registers a route handler for a specific HTTP method and URL path.
req — The incoming request object. Contains: req.params (URL params like :id), req.body (POST body), req.headers, req.query (query string).
res — The outgoing response object. Methods: res.send() (send text), res.json() (send JSON), res.status() (set status code).
app.listen(port, callback) — Starts the HTTP server on the given port.

Middleware:
Middleware is a function that runs between the incoming request and the route handler.
It has access to req, res, and a next() function that passes control to the next middleware or route.
app.use(express.json()) is built-in middleware that parses JSON request bodies automatically.
You can write custom middleware: 
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next(); // must call next() or the request will hang
});

How a Request Flows:
1. Client sends a request (e.g., GET /users/42)
2. Middleware runs first (e.g., parses the JSON body)
3. Express matches the URL and HTTP method to a route handler
4. The route handler runs and sends back a response

URL Parameters vs Query Strings:
URL parameter: /users/:id — accessed via req.params.id — used to identify a specific resource.
Query string: /users?sort=asc — accessed via req.query.sort — used for optional filters or options.

HTTP Status Codes in Express:
res.status(200).json({}) — OK (default for successful GET)
res.status(201).json({}) — Created (use after successfully creating a resource with POST)
res.status(400).json({}) — Bad Request (invalid input from client)
res.status(404).json({}) — Not Found (resource doesn't exist)
res.status(500).json({}) — Internal Server Error (something went wrong on the server)

Testing Routes:
You can test your Express routes from the terminal without a browser:
# GET request
curl http://localhost:3000/users/42

# POST request with JSON body
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Bob","email":"bob@example.com"}'
